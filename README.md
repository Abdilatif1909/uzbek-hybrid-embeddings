# Uzbek Hybrid Embeddings

Project to build and evaluate hybrid embedding models for the Uzbek language. The repository provides a reproducible pipeline to preprocess a raw Uzbek corpus, train Word2Vec and FastText embeddings, generate mBERT sentence embeddings, run intrinsic evaluations, and run a simple downstream text-classification benchmark.

## Project description

This codebase implements and compares three embedding approaches for Uzbek:
- Word2Vec (gensim) — static word embeddings
- FastText (gensim) — subword-aware embeddings
- mBERT — contextual sentence embeddings (mean pooled)

All scripts are designed to be runnable from the command line and read hyperparameters from config.yaml to improve reproducibility.

## Corpus description

Place all raw UTF-8 text files (*.txt) into the data/raw/ directory. The preprocessing script will:
- load all .txt files in data/raw/
- clean and normalize Uzbek text while preserving Latin and Cyrillic characters
- split documents into sentences using punctuation heuristics
- save a processed sample to data/processed/corpus.txt and the full processed corpus to data/processed/corpus_big.txt

Do not commit large raw corpora to the repository; keep them locally or provide a download link in the paper.

## Directory structure

- data/raw/            # put raw .txt files here
- data/processed/      # generated by preprocessing
- models/              # trained models and embeddings
- results/tables/      # CSV result tables
- results/figures/     # generated figures
- src/                 # scripts

## Configuration

Edit config.yaml to change hyperparameters. The following keys are used by scripts:
- vector_size, window, min_count, epochs, sg, min_n, max_n, workers, seed, sample_limit

## Quick start / Training instructions

Install dependencies (recommended in a virtual environment):

pip install -r requirements.txt

Run the pipeline (order matters):

python src/preprocessing.py
python src/train_word2vec.py
python src/train_fasttext.py
python src/generate_mbert_embeddings.py
python src/evaluate.py
python src/text_classification_benchmark.py

Each script prints progress and saves outputs under models/ and results/.

## Evaluation instructions

- Intrinsic evaluation: src/evaluate.py computes vocabulary sizes, OOV coverage, and average cosine similarity for predefined test words and saves results to results/tables/embedding_results.csv. It also saves a comparison figure to results/figures/.
- mBERT similarities are saved to results/tables/mbert_results.csv by src/generate_mbert_embeddings.py.
- Downstream benchmark results are saved to results/tables/downstream_results.csv by src/text_classification_benchmark.py.

## Hardware requirements

- CPU: 4+ cores recommended for faster training of Word2Vec and FastText
- RAM: depends on corpus size; 8GB+ recommended for moderate corpora
- GPU (optional): recommended for computing mBERT embeddings (transformers + torch). On CPU mBERT will be much slower.

## Library versions (tested)

- Python 3.8+
- numpy >= 1.22
- pandas >= 1.3
- scikit-learn >= 1.0
- gensim >= 4.1
- torch >= 1.10 (if using GPU for mBERT)
- transformers >= 4.15
- matplotlib, seaborn

Exact requirements are listed in requirements.txt. Pin versions as needed for reproducibility.

## Reproducibility

- All scripts read config.yaml. Set `seed` in config.yaml to fix randomness for training/evaluation splits and mBERT sampling.
